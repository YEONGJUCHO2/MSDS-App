# MSDS Review Lifecycle Revision

작성일: 2026-05-01  
대상 브랜치: `codex/msds-review-lifecycle`  
대상 독자: 제품 기획, 개발, QA, UI/UX 디자이너

## 1. 리비전 목적

이번 리비전은 MSDS 앱을 단순 업로드/추출 도구에서 현장 운영 흐름에 맞는 MSDS 관리 도구로 확장한다.

핵심 변경은 다음이다.

- 최초 등록된 MSDS는 검수 완료 상태로 시작한다.
- 공식 API 재조회에서 새 규제 해당 정보가 생기면 문서 단위로 검수 필요 상태가 된다.
- 검수 필요 상태는 버튼으로 임의 완료할 수 없고, 새 MSDS 재첨부가 완료되어야 검수 완료로 돌아간다.
- 대시보드와 MSDS 메뉴의 문서 목록 조작은 같은 기준으로 동작한다.
- 현장관리에서 현장별 MSDS 연결, 조회, 첨부 열기, 검수 필요 경고를 제공한다.

## 2. 업무 흐름 기준

### 2.1 최초 등록

보건관리자가 MSDS 파일을 업로드한다.

지원 파일:

- PDF
- Excel, `.xlsx`
- Word, `.docx`
- CSV

최초 등록된 문서는 보건관리자가 검수된 자료를 올린 것으로 보고 `검수 완료` 상태로 등록한다. 업로드 직후 문서 텍스트, 기본 정보, 구성성분, 사내 입력 포맷 후보가 생성된다.

### 2.2 정기/수동 재조회

문서에 포함된 구성성분은 공식 API 기준으로 재조회할 수 있다.

재조회 결과 기존 문서에 없던 규제 해당 표시가 새로 생기면 해당 MSDS는 문서 단위로 `검수 필요`가 된다.

검수 필요 상태에서 사내 입력 포맷의 새 규제 해당 값은 빨강색 볼드체로 표시된다. 같은 문서가 새 MSDS 재첨부로 검수 완료 상태가 되면 해당 강조는 일반 표시로 돌아간다.

### 2.3 검수 필요 해소

검수 필요 상태는 다중 선택 후 검수 완료 같은 일괄 버튼으로 해소하지 않는다.

해소 조건:

- 해당 문서에 새 MSDS 파일을 재첨부한다.
- 서버가 기존 추출 데이터, 성분, 검수 큐, 규제 매칭, 기본 정보를 해당 문서 범위에서 재처리한다.
- 재처리 완료 후 문서 상태를 `검수 완료`로 변경한다.

### 2.4 현장관리

현장 담당자는 현장관리 화면에서 여러 MSDS를 선택해 특정 현장명에 연결할 수 있다.

현장관리에서 제공해야 하는 정보:

- 현장별 연결된 MSDS 수
- 현장별 검수 필요 수
- 현장별 개정 필요 수
- 연결된 MSDS 파일 열기
- 검수 필요 MSDS 열기 전 경고 알림

검수 필요 MSDS를 열 때 표시되는 문구:

```text
해당 MSDS는 개정이 필요합니다. 현장 비치 필요시 보건담당자와 협의 바랍니다
```

## 3. 데이터 상태 모델

문서 단위 상태가 추가되었다.

| 필드 | 의미 |
|---|---|
| `review_state` | `approved` 또는 `needs_review` |
| `review_reason` | 검수 필요로 전환된 이유 |
| `review_required_at` | 검수 필요가 된 시각 |
| `review_completed_at` | 재첨부 후 검수 완료가 된 시각 |
| `last_regulatory_checked_at` | 마지막 공식 재조회 시각 |
| `replacement_uploaded_at` | 마지막 재첨부 시각 |

`review_queue`는 여전히 성분 추출 품질, CAS 누락, 수동 확인 필요 항목에 사용한다. 단, 대시보드와 현장관리의 검수 필요 수는 문서 단위 `review_state`를 우선 기준으로 한다.

## 4. API 변경

| 메서드/경로 | 목적 |
|---|---|
| `GET /api/documents` | 문서 목록과 문서 단위 검수 상태 조회 |
| `PATCH /api/documents/:documentId` | 문서 파일명 변경 |
| `GET /api/documents/:documentId/file` | 첨부파일 열기. PDF는 inline, 기타 파일은 다운로드 |
| `POST /api/documents/:documentId/replacement` | 검수 필요 문서에 새 MSDS 재첨부 |
| `POST /api/documents/recheck` | 선택 MSDS 공식 API 조회/재조회 |
| `GET /api/products` | 현장관리 연결 목록과 문서 검수 상태 포함 조회 |

## 5. 화면별 변경 요약

### 5.1 대시보드

대시보드의 최근 MSDS 목록은 MSDS 메뉴와 같은 기준으로 동작한다.

포함 기능:

- MSDS 검색
- 등록일 조회 버튼
- 등록 시작일/등록 종료일 선택
- 최대 6개월 기간 제한
- 검수 상태 필터: 전체, 검수 완료, 검수 필요
- 다중 선택
- 선택 조회/재조회
- 선택 첨부 다운로드
- 선택항목 삭제
- 업로드 이동
- 파일명 옆 연필 버튼으로 이름 수정
- 첨부파일 열기
- 검수 필요 문서 재첨부

검수 필요 카드 숫자는 기존 검수 큐 수가 아니라 문서 단위 `review_state = needs_review` 수를 표시한다.

### 5.2 MSDS 메뉴

MSDS 메뉴는 문서 목록과 상세 검수 영역을 한 화면에서 다룬다.

상단 목록:

- 대시보드와 같은 검색/날짜/상태 필터
- 같은 파일명 수정 방식
- 같은 첨부 열기 방식
- 검수 필요 문서에만 재첨부 노출

상세 영역:

- 물품 기본 정보
- 사내 입력 포맷
- 추출 근거
- 공식 조회/재조회
- 직접 수정/추가/삭제

### 5.3 현장관리

현장관리 화면은 업로드된 MSDS와 현장명을 연결하고, 현장별로 연결된 MSDS를 조회한다.

주요 영역:

- MSDS 선택 영역
- 사용현장 입력 영역
- 현장별 조회 영역
- MSDS별 연결 현황 영역

검수 필요 MSDS는 첨부파일을 열기 전에 경고 알림을 표시한다.

### 5.4 삭제된 메뉴

`개정 비교` 메뉴는 현재 업무 흐름에서 별도 메뉴로 유지할 이유가 낮아 제거했다.

개정 관련 동작은 문서 단위 검수 필요 상태, 재조회, 재첨부 흐름으로 흡수한다.

## 6. 추출 품질 리비전

CA-90R Excel 문서에서 다음 문제가 확인되었다.

- 공급사 값이 이상하게 추출됨
- CAS No.가 일부 누락됨
- 구성성분 행이 `54`, `55`, `56` 같은 숫자로 추출됨

원인:

- XLSX 내부 shared string 참조가 실제 문자열로 복원되지 않았다.
- 이 때문에 셀 값이 `산화 마그네슘`, `1309-48-4`, `80~90` 대신 shared string index 숫자로 남았다.

조치:

- XLSX 행 추출 단계에서 numeric-looking shared string 참조를 실제 문자열로 복원한다.
- 같은 행에서 중복된 물질명 셀은 인접 중복값으로 제거한다.
- 전화번호, CAS No., 함유량 범위 같은 주요 값도 복원 대상에 포함한다.
- `가. 제품명`, `나. 제품의 권고 용도...` 같은 KOSHA 스타일 섹션 라벨을 기본 정보 추출에서 처리한다.

확인된 개선 결과:

```text
공급사: 포스코퓨처엠
제조사: 포스코퓨처엠
제품명: CA-90R
용도: 내화/방연제
산화 마그네슘: 1309-48-4 / KE-22728 / 80~90
산화 알루미늄: 1344-28-1 / KE-01012 / 1~10
비정질 석영 유리: 60676-86-0 / KE-30959 / 1~5
```

다음 항목은 기능 구현보다 운영 정책 또는 디자인 상세 결정이 필요하다.
