# Designer Handoff: MSDS Review Lifecycle

작성일: 2026-05-01  
대상: UI/UX 디자이너  
범위: MSDS 문서 등록, 검수 상태, 재조회, 재첨부, 현장관리 화면

## 1. 문서 목적

이 문서는 디자이너가 화면 설계와 인터랙션 정리를 시작할 수 있도록 현재 제품의 화면 구성, 기능 단위, 상태, 컴포넌트, 사용자 액션을 정리한다.

이 문서는 시각 스타일 방향을 지정하지 않는다.

포함하지 않는 항목:

- 색상 팔레트 제안
- 폰트 제안
- 카드/테이블/버튼 스타일 제안
- 화면 미감 방향
- 브랜딩 톤

포함하는 항목:

- 화면별 정보 구조
- 필요한 사용자 액션
- 상태와 조건
- 컴포넌트 단위
- 알림/피드백 문구
- 예외/빈 상태

## 2. 주요 사용자

| 사용자 | 주요 목적 |
|---|---|
| 보건관리자 | MSDS 등록, 추출값 확인, 공식 재조회, 검수 필요 문서 재첨부 |
| 현장 담당자 | 자신이 사용하는 현장의 MSDS 확인, 첨부파일 열기 |
| 안전보건 관리자 | 현장별 MSDS 상태와 검수 필요 현황 확인 |

## 3. 전역 내비게이션

현재 주요 메뉴:

- 대시보드
- 업로드
- MSDS
- 물질정보
- 현장관리

제거된 메뉴:

- 개정 비교

전역 내비게이션에서 각 메뉴는 사용자가 현재 위치를 인지할 수 있어야 한다.

## 4. 공통 상태 정의

### 4.1 문서 검수 상태

| 상태 | 사용자 표시 | 의미 |
|---|---|---|
| `approved` | 검수 완료 | 현재 시스템상 현장 사용 가능한 MSDS |
| `needs_review` | 검수 필요 | 법규 변경 또는 재조회 결과로 새 MSDS 확인/재첨부 필요 |
| `unknown` | 분석 필요 | 추출 결과가 없거나 문서 분석이 부족한 상태 |

### 4.2 상태별 동작

| 조건 | 필요한 표시/동작 |
|---|---|
| 검수 완료 | 일반 상태 표시 |
| 검수 필요 | 문서 목록에서 검수 필요 표시 |
| 검수 필요 + 사내 입력 포맷 | 새 규제 해당 값은 강조 표시 |
| 검수 필요 + 첨부 열기 | 경고 알림 후 파일 열기 |
| 검수 필요 + MSDS 목록 | 재첨부 액션 노출 |
| 재첨부 완료 | 검수 완료로 전환, 강조 해제 |

## 5. 공통 문서 목록 컴포넌트

대시보드와 MSDS 화면은 같은 문서 목록 패턴을 사용한다.

### 5.1 목록 정보

각 행에 필요한 정보:

- 선택 체크박스
- 번호
- 파일명
- 파일명 전체보기용 tooltip 또는 동등한 풀네임 확인 수단
- 파일명 수정 진입 버튼
- 등록일자
- 검수 상태
- 첨부파일 열기
- 상세 보기 또는 문서 선택
- 삭제
- 검수 필요 상태일 때 재첨부

### 5.2 목록 상단 컨트롤

필요 컨트롤:

- MSDS 검색
- 등록일 조회 버튼
- 등록 시작일
- 등록 종료일
- 검수 상태 필터
- 현재 페이지 전체 선택
- 선택 개수 표시
- 선택 조회/재조회
- 선택 첨부 다운로드
- 선택항목 삭제

등록일 조회 조건:

- 날짜 필드는 기본적으로 숨겨져 있다.
- 사용자가 등록일 조회 버튼을 누르면 등록 시작일/등록 종료일 선택 영역이 열린다.
- 조회 기간은 시작일과 종료일 사이 최대 6개월이다.

### 5.3 파일명 수정 인터랙션

기본 상태:

- 파일명은 읽기 상태로 보인다.
- 파일명 옆에 이름 수정 진입 버튼이 있다.

수정 상태:

- 수정 입력 필드가 열린다.
- 저장 버튼이 함께 표시된다.
- 저장 후 입력 필드는 닫힌다.
- 빈 값 또는 기존과 같은 값은 저장하지 않는다.

### 5.4 첨부파일 인터랙션

첨부 액션:

- 첨부파일 표시 아이콘과 텍스트가 함께 있다.
- PDF는 브라우저에서 열릴 수 있다.
- Excel, Word, CSV 등은 브라우저 정책에 따라 다운로드될 수 있다.

검수 필요 문서의 현장관리 첨부 열기:

```text
해당 MSDS는 개정이 필요합니다. 현장 비치 필요시 보건담당자와 협의 바랍니다
```

## 6. 대시보드

### 6.1 목적

전체 MSDS 현황을 빠르게 보고 최근 문서 관리 액션으로 이동한다.

### 6.2 화면 구성

상단 지표:

- 등록된 MSDS
- 검수 필요
- 등록된 화학물질
- 개정 필요

최근 MSDS 영역:

- 공통 문서 목록
- 업로드 버튼
- 검색/날짜/상태 필터
- 선택 액션

### 6.3 필요한 상호작용

- 지표 클릭 시 관련 화면으로 이동
- 업로드 버튼 클릭 시 업로드 화면으로 이동
- 검색어 입력 시 목록 필터
- 등록일 조회 버튼 클릭 시 날짜 필터 노출
- 상태 필터 변경 시 목록 필터
- 선택 조회/재조회 클릭 시 선택 문서 재조회
- 선택 첨부 다운로드 클릭 시 선택 문서 파일 열기/다운로드
- 선택항목 삭제 클릭 시 삭제 확인
- 파일명 수정 버튼 클릭 시 수정 입력 표시
- 첨부 클릭 시 파일 열기
- 보기 클릭 시 MSDS 상세 화면으로 이동

### 6.4 빈 상태

조건:

- 등록된 문서가 없음
- 검색/필터 결과가 없음

필요 정보:

- 현재 표시할 MSDS가 없다는 메시지
- 업로드로 이어질 수 있는 액션

## 7. 업로드

### 7.1 목적

보건관리자가 MSDS 파일을 등록한다.

### 7.2 화면 구성

필요 요소:

- 파일 드롭 영역
- 파일 선택 입력
- 지원 형식 안내
- 업로드 진행 상태
- 업로드 성공/실패 결과
- 파일별 처리 결과

### 7.3 상태

| 상태 | 필요한 정보 |
|---|---|
| 기본 | 파일 선택 가능 |
| 드래그 중 | 드롭 가능 상태 |
| 업로드 중 | 처리 중인 파일 수 |
| 일부 성공 | 성공/실패 파일 구분 |
| 실패 | 실패 이유 |

## 8. MSDS 화면

### 8.1 목적

등록된 MSDS를 관리하고, 선택한 문서의 기본 정보와 사내 입력 포맷을 확인/수정한다.

### 8.2 화면 구성

상단:

- 공통 문서 목록

하단 또는 상세 영역:

- 물품 기본 정보
- 사내 입력 포맷
- 추출 근거 보기

### 8.3 물품 기본 정보

필드:

- 공급사
- 제조사
- 대표전화
- E-mail
- 제품명
- 용도
- ITEM코드
- MSDS번호
- 제조구분
- 사업장
- 최종개정일자
- 구분
- 검토자
- 검토의견

필요 액션:

- 필드 값 수정
- 저장

### 8.4 사내 입력 포맷

행 정보:

- CAS No.
- 화학물질명
- MIN
- MAX
- 단일
- 특별관리물질
- 관리대상유해물질
- 허가대상물질
- 제조금지물질
- PSM 등 사내 입력 대상 컬럼

필요 액션:

- 행 추가
- 행 수정
- 행 삭제
- 공식 조회/재조회
- 전체 복사

상태:

- 검수 완료 문서: 일반 표시
- 검수 필요 문서: 새 규제 해당 값 강조 표시

### 8.5 추출 근거

필요 정보:

- 추출된 행의 원문 근거
- 공식 조회 결과 메시지
- 수동 검토 필요 사유

## 9. 물질정보

### 9.1 목적

사업장 기준 화학물질과 공식 조회 결과를 관리한다.

### 9.2 필요한 정보

- CAS No.
- 화학물질명
- 연결된 MSDS 또는 문서 수
- 공식 조회 상태
- 규제 해당 정보
- 마지막 조회일

### 9.3 필요한 액션

- 검색
- 상태 필터
- 재조회
- 연결된 문서 확인

## 10. 현장관리

### 10.1 목적

현장별로 비치해야 하는 MSDS를 연결하고 조회한다.

### 10.2 화면 구성

MSDS 연결 영역:

- MSDS 검색
- 검수 상태 필터
- MSDS 다중 선택 목록
- 선택 개수
- 사용현장 입력
- 선택 MSDS를 현장에 묶기

현장 관리 조회:

- 현장 선택 목록
- 현장별 MSDS 수
- 현장별 개정 필요 수
- 현장별 검수 필요 수

MSDS별 연결 현황:

- 검색
- 상태 필터
- 제품명 또는 문서명
- 파일명
- 공급사/제조사
- 현장명
- 상태
- 첨부 열기
- 연결 삭제

### 10.3 상태 필터

필터 옵션:

- 전체
- 검수 완료
- 검수 필요
- 개정 필요
- MSDS 미연결

### 10.4 필요한 상호작용

- MSDS 다중 선택
- 사용현장 입력
- 선택 MSDS 현장 연결
- 현장 선택
- 검색어 입력
- 상태 필터 변경
- 첨부파일 열기
- 검수 필요 문서 경고 알림
- 연결 삭제 확인

### 10.5 빈 상태

필요 빈 상태:

- 업로드된 MSDS 없음
- 검색된 MSDS 없음
- 현장 연결 없음
- 선택한 현장에 표시할 MSDS 없음

## 11. 피드백과 확인 메시지

필요 피드백:

- 업로드 중
- 업로드 완료
- 업로드 실패
- 조회/재조회 중
- 조회/재조회 완료
- 조회/재조회 실패
- 이름 저장 완료 또는 실패
- 재첨부 처리 중
- 재첨부 완료 또는 실패
- 현장 연결 완료 또는 실패
- 삭제 확인

현재 확인이 필요한 위험 액션:

- 문서 삭제
- 선택 문서 삭제
- 성분 행 삭제
- 현장관리 연결 삭제

## 12. 접근성과 사용성 계약

디자인 결과물에서 유지해야 하는 동작 계약:

- 파일명은 길어도 전체 이름을 확인할 수 있어야 한다.
- 키보드 사용자가 검색, 필터, 파일명 수정, 첨부 열기, 재첨부, 삭제에 접근할 수 있어야 한다.
- 상태는 색상만으로 구분되지 않아야 한다.
- 검수 필요 문서의 경고 문구는 사용자가 읽고 판단할 수 있어야 한다.
- 삭제와 재첨부는 서로 혼동되지 않아야 한다.
- 대시보드와 MSDS 화면의 문서 목록 조작은 같은 방식으로 이해되어야 한다.

## 13. 디자이너에게 전달할 데이터 예시

문서 목록 행 예시:

```json
{
  "documentId": "doc-1",
  "fileName": "복사본 부정형 2_CA-90R_고유번호.xlsx",
  "uploadedAt": "2026-04-30T00:00:00.000Z",
  "reviewState": "approved",
  "componentCount": 3,
  "queueCount": 6
}
```

검수 필요 문서 예시:

```json
{
  "documentId": "doc-2",
  "fileName": "sample-msds.pdf",
  "uploadedAt": "2026-04-30T00:00:00.000Z",
  "reviewState": "needs_review",
  "reviewReason": "official_regulatory_change",
  "componentCount": 8,
  "queueCount": 0
}
```

현장관리 연결 예시:

```json
{
  "productName": "CA-90R",
  "documentFileName": "복사본 부정형 2_CA-90R_고유번호.xlsx",
  "supplier": "포스코퓨처엠",
  "manufacturer": "포스코퓨처엠",
  "siteNames": "2STS",
  "documentReviewState": "approved"
}
```

## 14. 설계 시 확인이 필요한 제품 질문

다음은 디자인 과정에서 제품 담당자와 확인할 질문이다.

- 현장관리에서 현장 수가 많아질 때 현장 선택 방식은 어떤 탐색 구조가 필요한가?
- 검수 필요 MSDS가 현장에 이미 연결된 경우 현장 담당자에게 어떤 수준의 알림이 필요한가?
- 재첨부 이력은 같은 화면에서 보여야 하는가, 별도 이력 화면이 필요한가?
- 공식 API 재조회가 오래 걸릴 때 사용자가 화면을 떠나도 진행 상태를 볼 필요가 있는가?
- 모바일 또는 태블릿 현장 사용이 중요한가?
- 현장 담당자는 편집 권한 없이 조회/열기만 가능한 역할이 필요한가?
